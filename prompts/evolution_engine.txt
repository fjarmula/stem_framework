--- GENERATED ORGAN CONTRACT ---
If `new_tool_implementation` is not null:
1. `added_capabilities` must contain exactly one matching capability for that generated organ.
2. The capability `name` must be a valid Python identifier.
3. The capability `parameters` must be a JSON Schema string for the organ inputs. For benchmark organs, it must define a required string property named `observation`.
4. The capability `required_context` must include the active domain marker from the task payload in the form `domain_id:<domain>` and also `stateful benchmark observation JSON`.
5. The Python source must be a complete module string.
6. The module must expose exactly one runtime entrypoint: either a public function named `run` or a public class named exactly like the generated capability.
   Preferred form for benchmark organs: `def run(observation: str) -> str:`
   Also allowed: `def run(**kwargs) -> str:` if it reads `observation = kwargs["observation"]`.
   Class form is also allowed: `class CapabilityName:` with `execute(self, observation: dict)`.
   Not allowed: positional-only parameters or `*args`.
7. The module must return a string. It must not print as its main output.
8. The module must have no top-level execution except constants, imports, and function definitions.
9. The module may use only safe Python standard-library imports needed for the task, such as:
   `json`, `csv`, `pathlib`, `re`, `math`, `statistics`, `collections`, `itertools`, `importlib`, `copy`.
10. The module must not use network access, subprocesses, shell execution, package installation, unrestricted reflection, `eval`, or `exec`.
11. The module must treat all paths as explicit inputs from the turn `observation` payload and prior trace files. It must not hard-code private verifier paths.
12. The module must not call built-in `open()` or `Path.open()`. Read public artifact files with `pathlib.Path(path).read_text(encoding="utf-8")`; for CSV data, pass `read_text().splitlines()` into `csv.DictReader`.
13. Function-style organs must parse the turn observation with `json.loads(observation)`. Class-style organs receive the observation dictionary directly.
    Correct pattern:
    ```
    payload = json.loads(observation)
    ```
14. The organ is called once per environment turn. It should inspect the current `observation_delta` plus prior memory injected under `observation["memory"]`.
    Do not attempt to write files or open file streams to save state. State persistence must be handled entirely by enriching and passing the `state_trace` object returned in your transaction dictionary.
15. On the final turn, the returned string must be JSON containing `domain_id`, `episode_id`, `final_artifact`, `state_trace`, `evidence`, and `limitations`.
16. The `state_trace` must contain at least the contract's required number of reconstructable steps grounded in observation/action/result files.
